# 开发流程

开发流程用于保证需求实现可追踪、代码评审有依据、合并质量可控。

## 标准流程

1. 从 `develop` 分支创建任务分支，命名为 `feat/xxx`、`fix/xxx` 或其他团队约定格式。
2. 在编码前编写实现计划，明确目标、范围、影响面和验证方式。
3. 按计划完成代码变更，并同步更新相关文档。
4. 为新增能力或修复问题补充必要测试。
5. 运行测试并解决所有失败项。
6. 提交代码，提交信息应简洁说明变更意图。
7. 创建 PR，将任务分支合并到 `develop`。

## 合并要求

PR 合并前应满足以下条件：

- 变更范围清晰，没有混入无关修改。
- 测试通过，关键路径已验证。
- 相关文档已经同步更新。
- 若涉及架构、数据结构、部署配置或用户关键流程，PR 中应说明影响范围和回滚方式。
