# 部署规范

部署以环境隔离、分支映射清晰、发布过程可追踪、异常可回滚为基本原则。

## 环境划分

- 生产环境：用户实际使用的环境，对应仓库的 `main` 分支。
- 预览环境：用于合并前验证，除数据隔离外应尽量与生产环境保持一致，对应仓库的 `develop` 分支。
- 本地环境：用于日常开发和调试，对应从 `develop` 检出的任务分支。

## 发布原则

1. 生产发布只能来自已经验证过的稳定分支。
2. 发布前应确认测试、构建、环境变量和数据库迁移状态。
3. 涉及数据库、权限、支付、登录等高风险变更时，应准备回滚或降级方案。
4. 发布后应检查关键路径，确认服务、日志和核心功能状态正常。

## 配置管理

不同环境的配置应明确隔离。敏感信息不得写入仓库，必须通过环境变量或平台密钥管理能力维护。
