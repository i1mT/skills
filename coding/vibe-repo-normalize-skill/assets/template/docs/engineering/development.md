# 项目开发

项目开发以环境可复现、启动方式清晰、依赖管理一致为基本要求。

## 依赖安装

使用项目约定的包管理器安装依赖：

```bash
pnpm i
```

依赖版本应由锁文件统一管理。新增依赖前应确认其必要性、维护状态和对包体积或运行时的影响。

## 本地启动

本地开发使用以下命令启动：

```bash
pnpm run dev
```

启动前应确认环境变量、数据库连接和第三方服务配置已准备完成。默认情况下，本地环境应连接预览或专用开发资源，不直接连接生产资源。

## 开发原则

1. 保持单次变更范围清晰，避免在一个任务中混入无关重构。
2. 优先遵循项目已有目录结构、命名方式和抽象边界。
3. 新增代码应同步维护相关文档、测试和必要的文件头说明。
4. 遇到明显架构坏味道时，应先记录问题并确认是否纳入当前任务处理。
