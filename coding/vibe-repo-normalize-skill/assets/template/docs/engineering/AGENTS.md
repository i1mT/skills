# 工程文档

本目录用于保存项目长期维护所需的工程上下文，包括开发方式、测试策略、开发流程、部署环境和数据层约定。

- `development.md`：说明项目本地开发、依赖安装、启动方式和环境配置原则。
- `test.md`：说明测试目标、测试范围、执行要求和合并门槛。
- `workflow.md`：说明从需求到 PR 合并的标准开发流程。
- `deployment.md`：说明本地、预览、生产环境的职责边界和分支映射。
- `context.md`：说明 GEB 仓库上下文文档机制。
- `database.md`：说明数据库和 ORM 的基础选型。
