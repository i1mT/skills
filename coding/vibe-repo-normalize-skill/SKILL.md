# Vibe Coding 代码仓库规范化

将 `assets/template` 中的模板为基准，改造代码仓库。

## 规范

vibecoding 仓库的规范如下：
1. 根目录必须要有 `AGENTS.md`；以根目录 `AGENTS.md` 作为 Coding Agent 理解项目的入口。
2. 根目录必须要有 `docs` 文件夹，`docs` 文件夹是项目持久化文档系统的保存位置。
3. `docs` 下必须要有 `AGENTS.md`，它是整个持久化文档系统的导航目录。
4. `docs` 下面分了4个部分：
  a. design：视觉设计系统文档
  b. engineering: 技术实现文档
  c. plan: 全生命周期的计划文档
  d. product: 项目产品文档
5. 必须使用 GEB 文档管理机制

## 整体步骤

1. 检测当前代码仓库的目录结构，做预处理
2. 将 template 中的文件与文件夹写入到代码仓库
3. 理解当前代码仓库的产品、技术基本情况
4. 根据当前仓库的现状，更新文档
5. 检查文件夹、文件是否已规范化
6. 检查文件内容是否已经适配当前代码仓库
7. 结束后，将无关文件做后处理

## 1. 代码仓库预处理

首先获取当前代码仓库的文件目录结构，并预处理使得文件名、目录结构不与我们将写入的规范文件、目录冲突。

### 文件

1. 若根目录有 AGENTS.md，重命名为 AGENTS.md.backup。
2. 若根目录有 CLAUDE.md，重命名为 CLAUDE.md.backup。

### 文件夹

若根目录有 docs 文件夹，重命名为 docs-backup。

## 2. 写入规范模板

将 `assets/template` 下面的内容，复制到项目的根目录中。
由于第一步我们已经处理过了仓库中，所以这里不会有文件冲突。

## 3. 梳理当前仓库

你需要对当前仓库做一个全景扫描，理解仓库的产品、技术等方案，了解它解决的问题，使用的解决方案。
并且阅读之前的 AGENTS.md、CLAUDE.md，docs 文件夹下的文档等（若有）。

这一步是准备将仓库的现状，更新到我们刚刚写入的文档中。

## 4. 更新文档

根据上一步的梳理、总结，更新我们上面第2步写入的文档。
我们准备的文档中都有预先写好的模板，按照里面的结构进行更新即可。

## 5. 检查规范

根据 `规范` 章节中的定义，检查当前仓库是否已经符合规范。

## 6. 检查文件内容是否已更新

检查我们规范中的文档，是否已经修改为了当前仓库的文档，而不是最原始的模板内容。

## 7. 后处理

1. 将前面第一步重命名的 `CLAUDE.md.backup`、`AGENTS.md.backup`、`docs-backup` 全部移动到一个新文件夹 `docs-archive` 中。
2. 如果之前有 `CLAUDE.md` 文档，现在在根目录创建一个新的 CLAUDE.md 文档，内容是：
```
@AGENTS.md
```

# 结束
全部完成后，提示用户已经完成文档规范化处理。后续 vibe coding 会按照文档系统进行记录和持久化。
并告知当前文档系统现状。
